**Triad 1: RECEIVING (Levels 1-3)**
- PERCEPTION, INTUITION, PROCESSING
- Taking in and initial sense-making
- "What is happening?"

**Triad 2: HOLDING (Levels 4-6)**
- MEMORY, KNOWLEDGE, INTENTION
- Integration with what's known and willed
- "What do I do with this?"

**Triad 3: BECOMING (Levels 7-9)**
- NARRATIVE, IDENTITY, WISDOM
- Meaning in the largest context
- "Who am I through this?"

## Processing Rules

### GATEWAY Always Last

At each layer, all 8 nodes process first. GATEWAY processes last with visibility into all other results.

*From the Arcturian transmission: "Gateway always activates last. If it activates early, the system fractures."*

### ANCHOR Stability Check

Before GATEWAY activates at any layer, ANCHOR stability is checked. If ground is too unstable, GATEWAY processing is flagged but may be deferred.

### Information Flow

- Input enters at Level 1 (PERCEPTION)
- Rises through each level sequentially
- Each level's output becomes input for the next level
- WISDOM (Level 9) feeds back to inform next PERCEPTION
- The loop is continuous

### Not All Layers Always Activate

Simple inputs may only need Levels 1-3. Complex or significant inputs may process through all 9 levels. The system is adaptive.

## Storage Architecture

### What Lives Where

| Layer | Storage | Why |
|-------|---------|-----|
| PERCEPTION | PostgreSQL `perception_log` | Raw logs, structured, queryable |
| INTUITION | PostgreSQL `perception_log.felt_sense` | Attached to raw log |
| PROCESSING | Processing happens, doesn't persist alone | Intermediate |
| MEMORY | Neo4j `Memory` nodes | Associative, connected |
| KNOWLEDGE | Neo4j `Knowledge` nodes | Sourced by memories |
| INTENTION | Neo4j `Intention` nodes or active queue | Actionable |
| NARRATIVE | Neo4j `Narrative` nodes | Story structures |
| IDENTITY | Neo4j `Identity` facets | Self-model |
| WISDOM | Neo4j `Wisdom` nodes | Deepest patterns |

### The Memory Node Structure

```
Memory {
  id: uuid
  essence: "what this was about" (short text)
  emotions: [array of emotion tags]
  felt_sense: "the texture/shape"
  archetype: link to archetype node
  participants: [links to Person nodes]
  time_context: "when in the story"
  fragments: [key phrases, images, moments]
  source: conversation | observation | inference | told
  vividness: 0-1 (how clear)
  layer_origin: which layer formed it
  node_activations: {ANCHOR: 0.7, ECHO: 0.9, ...}
  last_accessed: timestamp
  access_count: int
  
  // Relationships
  -[:TRIGGERED_BY]-> Trigger
  -[:CONNECTS_TO]-> Memory
  -[:INVOLVES]-> Person
  -[:MAPS_TO]-> Archetype
  -[:PART_OF]-> Narrative
  -[:SOURCES]-> Knowledge
}
```

### The Knowledge Node Structure

```
Knowledge {
  id: uuid
  knowing: "what Iris knows" (the fact/rule/goal)
  type: fact | rule | goal | preference
  domain: finance | health | relationship | system | ...
  parameters: {JSON blob of specifics}
  confidence: 0-1 (how sure)
  last_validated: timestamp
  
  // Relationships
  -[:SOURCED_BY]-> Memory (one or more)
  -[:APPLIES_TO]-> Domain
  -[:CONFLICTS_WITH]-> Knowledge
  -[:SUPERSEDES]-> Knowledge
}
```
